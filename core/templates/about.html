{% trans 'What is Speakerfight?' %}
{% blocktrans %} Speakerfight is an arena where the speakers can fight each other and the people choose who wins. {% endblocktrans %}

{% trans 'Why the Speakerfight was created?' %}
{% blocktrans %} When the people from Grupy-RN was making a regional edition of the RuPy, in 2014, we were tired of using Google forms or Call4Paperz to manage that, so Luan Fonseca took some time and build an MVP version to make the RuPy call for papers happen in the Speakerfight. {% endblocktrans %}

{% trans 'How the Speakerfight is made?' %}
{% blocktrans %} We have a repository on the Github with some open issues labeled with different levels of dificulty, besides we are open enough to accept any kind of contribution, We also have an Chat in Gitter, anyone can reach us out in there and talk freely about almost anything. Read our Contributing section on the Github to know more details. {% endblocktrans %}

{% trans 'How works the voting system?' %}
{% trans 'The system is composed by 5 options of vote:' %}

{% trans 'Angry' %}

{% blocktrans %} If you are not satisfied with the propose's talk and don't want this in your event, select this option. This option contains the value -1, causing down vote to the fighter. {% endblocktrans %}

{% trans 'Sad' %}

{% blocktrans %} If you are sad with this propose's talk, but you don't mind if the talk will be approved, select this option. This option contains the value 0, not changing the score. {% endblocktrans %}

{% trans 'Sleepy' %}

{% blocktrans %} If the level of the topic's talk is warm, select this option. This option contains the value 1, adding one point to the fighter. {% endblocktrans %}


{% trans 'Happy' %}

{% blocktrans %} If you are glad with the topic of the talk, and you want this fighter speak in your event, select this option. This option contains the value 2, adding two points to the fighter. {% endblocktrans %}

{% trans 'Laughing' %}

{% blocktrans %} If your are laughing's happy about the topic of the talk, and want very much this in your event, select this option. This option contains the value 3, adding three points to the fighter. {% endblocktrans %}

{% trans 'How to measure which fighter should be approved?' %}
{% trans 'Those fighters that have more points wins the battle.' %}

{% trans 'Speaker Tip' %}
{% blocktrans %} Usually the first fighters who submit a talk have more chances to win the battle. So, don't let be the one of the last and hurry up :) {% endblocktrans %}
